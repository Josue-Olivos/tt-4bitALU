## How it works

This project implements a 4-bit Arithmetic Logic Unit (ALU). An ALU is a fundamental building block of processors and digital systems that performs arithmetic and logical operations on binary data.

The ALU accepts two 4-bit operands, A and B, and a 3-bit opcode that selects which operation to perform. The result is produced on a 4-bit output bus along with status flags.

### Inputs

- A[3:0] = First operand
- B[3:0] = Second operand
- OPCODE[2:0] = Operation select

### Supported Operations

| Opcode | Operation |
|----------|-----------|
| 000 | A + B |
| 001 | A - B |
| 010 | A AND B |
| 011 | A OR B |
| 100 | A XOR B |
| 101 | NOT A |
| 110 | A << 1 |
| 111 | A >> 1 |

### Outputs

- RESULT[3:0] = Result of the selected operation
- CARRY_OUT = Carry or borrow flag generated by arithmetic operations
- ZERO_FLAG = Asserted when the result equals zero

The design is purely combinational, meaning the outputs update immediately when the inputs or opcode change. No internal registers or memory elements are used.

## How to test

Apply values to the A and B input buses and select an operation using the opcode inputs.

### Example 1: Addition

Inputs:

- A = 0101 (5)
- B = 0011 (3)
- OPCODE = 000

Expected Outputs:

- RESULT = 1000 (8)
- CARRY_OUT = 0
- ZERO_FLAG = 0

### Example 2: Subtraction

Inputs:

- A = 1000 (8)
- B = 0011 (3)
- OPCODE = 001

Expected Outputs:

- RESULT = 0101 (5)

### Example 3: AND Operation

Inputs:

- A = 1100
- B = 1010
- OPCODE = 010

Expected Output:

- RESULT = 1000

### Example 4: XOR Operation

Inputs:

- A = 1111
- B = 0101
- OPCODE = 100

Expected Output:

- RESULT = 1010

### Example 5: Zero Flag

Inputs:

- A = 0101
- B = 0101
- OPCODE = 001

Expected Outputs:

- RESULT = 0000
- ZERO_FLAG = 1

## External hardware

No external hardware is required.

The design uses only the standard Tiny Tapeout input and output pins and can be tested directly on a Tiny Tapeout development board.
